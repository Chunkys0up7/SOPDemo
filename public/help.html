Frequently Asked Questions

How do I create a new SOP from a template? ‚ñº
Click the "+ New SOP" button in the navigation bar or select a template from the dashboard. Choose from Atom (reusable component), Molecule (multi-step workflow), Organism (complete workflow), or Complete SOP (production-ready). The template will load with all metadata fields pre-configured for searchability and reusability.
What's the difference between Atom, Molecule, and Organism templates? ‚ñº
These are based on Atomic Design principles: Atoms are single-purpose reusable components (e.g., password reset). Molecules combine multiple atoms into multi-step procedures (e.g., new user setup). Organisms orchestrate complete end-to-end workflows with roles, timelines, and escalation matrices (e.g., IT support workflow).
How do I track which SOPs need compliance approval? ‚ñº
Navigate to the Approvals page to see all pending SOP approvals. You can filter by compliance framework (SOX, HIPAA, GDPR, etc.) and priority level. Each SOP shows its compliance frameworks in the metadata, making it easy to track regulatory requirements.
Can I reuse parts of existing SOPs in new documents? ‚ñº
Yes! Atoms are designed for reusability. When viewing an SOP, you'll see which components it uses and where they're reused across other SOPs. The "usedIn" metadata field tracks all workflows that include each atom, making it easy to see dependencies.
How do I search for SOPs by department or complexity? ‚ñº
Use the faceted search on the Search page or Browse page. You can filter by department, process category, complexity level (Basic/Intermediate/Advanced), audience, compliance frameworks, and more. All enhanced templates include rich metadata to make SOPs highly searchable.
What happens when I update an SOP that's used in other workflows? ‚ñº
The system will show you an impact analysis listing all SOPs and workflows that reference the component you're updating. You'll see which sections are affected and can choose to cascade the update or create a new version. This ensures you don't break dependent workflows.
