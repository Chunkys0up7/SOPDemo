What is the SOP Management System?

The Pursuit Bank SOP Management System is an AI-powered platform for creating, organizing, and navigating Standard Operating Procedures (SOPs). It uses advanced technologies like semantic search, dependency graphs, and modular component architecture to make documentation easier to create, find, and maintain.

Why This System Exists

Traditional SOP management has several problems:

  • Hard to find: Searching by keywords often misses relevant procedures
  • Duplication: Same information repeated across multiple documents
  • Unknown dependencies: Changing one SOP can break others
  • Technical barriers: Creating SOPs requires knowledge of complex formats and metadata

Our system solves these with:

  • Semantic Search (RAG): Find SOPs by asking questions in natural language
  • Dependency Graphs: See how SOPs connect and what breaks if you change something
  • Atomic Design: Reusable components eliminate duplication
  • AI Assistance: System generates all technical details automatically

Quick Start Guide

Start at the Dashboard
See stats, recent SOPs, and quick actions. Use the search bar to find procedures or click "My Workspace" to browse all SOPs.
Browse SOPs in Workspace
The Workspace shows all SOPs in a 3-panel layout. Click any SOP to see its full details, dependencies, and reusable components. Use the search box to filter by name or department.
View Dependencies in Graph
The Graph Viewer shows how SOPs connect. Click nodes to see details. Change layouts (Hierarchical, Organic, Circle, Grid) and filter by type (SOPs, Atoms, Molecules, Organisms).
Create New SOPs
Click "Contribute" to create a new SOP. Just write the procedure in plain language - the AI generates all metadata, compliance tags, and technical formatting automatically.
Use Semantic Search
Instead of exact keywords, ask questions like "How do I verify FHA credit requirements?" The system uses AI to find relevant SOPs even when your words don't match exactly.

Glossary of Terms

SOP (Standard Operating Procedure)
A documented step-by-step process that describes how to perform a specific task or operation. SOPs ensure consistency and compliance across the organization.
Example: "FHA Underwriting Guidelines" is an SOP that describes how to underwrite FHA loans.
RAG (Retrieval-Augmented Generation)
An AI technique that combines search with language understanding. Instead of matching exact keywords, RAG understands the meaning of your question and finds relevant answers across all documents.
Example: Asking "What's the minimum credit score for FHA?" will find the answer even if the SOP says "FICO requirements" instead of "credit score."
Semantic Search
Search that understands meaning and context, not just exact word matches. It uses AI to find documents related to your query even when different terminology is used.
Example: Searching "close a loan" will also find SOPs about "clear to close" and "closing procedures."
Dependencies
Relationships between SOPs where one procedure requires or references another. Understanding dependencies is critical to avoid breaking workflows when making changes.
Example: "Clear to Close" depends on "FHA Underwriting" because you must complete underwriting before clearing a loan.
Impact Analysis
Assessment of what will be affected if you change or delete an SOP. Shows all dependent procedures and components that reference the SOP.
Example: Changing "Credit Report Analysis" impacts 3 other SOPs that depend on it.
Atomic Design (Atoms, Molecules, Organisms)
A modular approach where complex procedures are built from smaller, reusable components. Atoms are basic elements (forms, checklists). Molecules combine atoms into simple processes. Organisms are complete workflows.
Example: "Credit Score Form" (atom) + "DTI Calculator" (atom) = "Financial Verification" (molecule) ‚Üí "Complete Underwriting" (organism)
Atoms
The smallest reusable building blocks - individual forms, templates, checklists, or data points that can't be broken down further. Used across multiple SOPs.
Example: "Credit Score Verification Form", "I-9 Employment Form", "Expense Report Template"
Molecules
Simple processes created by combining multiple atoms. Molecules perform a single function but require multiple steps or components.
Example: "Employment Verification Workflow" combines resume review, reference checks, and background check forms.
Organisms
Complex, complete workflows that combine atoms and molecules into full end-to-end processes. These represent entire procedures from start to finish.
Example: "Complete Loan Underwriting Workflow" includes credit checks, income verification, appraisal, and final approval steps.
Ontology
A formal structure that defines relationships between concepts. Our ontology maps how SOPs, departments, regulations, and processes connect to each other.
Example: The ontology knows that "FHA" is a type of "Mortgage Product" regulated by "HUD" and requires "Credit Verification."
Compliance Framework
A set of regulations, standards, or guidelines that procedures must follow. Examples include FHA rules, TRID requirements, SOX regulations, and RESPA compliance.
Example: An FHA underwriting SOP must comply with HUD requirements for credit scores, DTI ratios, and appraisals.
Vector Embeddings
Mathematical representations of text that capture meaning. Similar concepts have similar embeddings, allowing the system to find related SOPs even with different wording.
Example: "Credit score" and "FICO rating" have similar embeddings, so searching for one finds the other.
Dependency Graph
A visual representation showing how SOPs connect to each other and their components. Nodes are SOPs/components, edges show dependencies or relationships.
Example: An arrow from "FHA Underwriting" to "Credit Analysis" means underwriting depends on credit analysis.
Process Generation
AI-assisted creation of step-by-step workflows based on existing SOPs. The system analyzes dependencies and components to build custom procedures for specific scenarios.
Example: Generate a checklist for "New Mortgage Processor Onboarding" from existing HR and training SOPs.
Metadata
Information about an SOP beyond its content - version number, owner, department, compliance tags, last updated date, status, etc. Used for search and organization.
Example: Version 2.1.0, Owner: Mortgage Operations, Status: Active, Last Updated: 2025-11-12

Key Features Explained

Dashboard

The home page shows an overview of your SOP ecosystem:

  • Stats: Total SOPs, recent updates, compliance rate, pending reviews
  • Quick Actions: Fast access to Workspace, Graph, Create SOP, and Search
  • Recent SOPs: Most recently accessed or updated procedures
  • Activity Feed: Changes, approvals, and updates across the system

Workspace

Your main working area with 3 panels:

  • Left Panel: Browse and search all SOPs
  • Center Panel: View full SOP details, steps, and components
  • Right Panel: See dependencies and quick actions

Click any SOP to see its purpose, step-by-step procedure, reusable components, and which other SOPs depend on it or that it depends on.

Graph Viewer

Interactive visualization of SOP relationships:

  • Layouts: Hierarchical (default), Organic, Circle, Grid
  • Filters: Show only SOPs, Atoms, Molecules, or Organisms
  • Click nodes: View details and highlight connections
  • Impact analysis: See how many items will be affected by changes
  • Color coding: Blue (SOPs), Green (Atoms), Orange (Molecules), Purple (Organisms)

Semantic Search

RAG-powered search that understands meaning:

  • Ask questions in natural language: "What are FHA credit requirements?"
  • Find SOPs even with different terminology
  • Get relevance scores showing how well results match your query
  • See context highlighting showing where your answer is found

AI-Assisted Contribution

Creating SOPs is simple:

  • Write your procedure in plain language
  • AI automatically generates all metadata, tags, and formatting
  • AI identifies compliance requirements and frameworks
  • AI suggests reusable components from existing SOPs
  • Review and submit - no technical knowledge required

Common Workflows

Finding an Existing SOP

  1. Use the search bar on the Dashboard or go to Workspace
  2. Type keywords or ask a question in natural language
  3. Click the SOP to view full details
  4. Review dependencies in the right panel

Creating a New SOP

  1. Click "Contribute" or "Create SOP" from Dashboard
  2. Enter basic information: title, department, purpose
  3. Write procedure steps in your own words
  4. Add notes about who uses it and when
  5. Click "Generate SOP with AI" - the system handles the rest
  6. Review generated metadata and approve

Understanding SOP Dependencies

  1. Open the SOP in Workspace
  2. Check the Dependencies section in the right panel
  3. Click "View in Graph" to see visual relationships
  4. Review Impact Analysis to see what breaks if you change it

Checking Compliance Coverage

  1. View the SOP metadata in Workspace
  2. Check compliance tags (FHA, TRID, RESPA, SOX, etc.)
  3. Review Dashboard stats for overall compliance rate
  4. Use semantic search to find gaps: "Are there SOPs for HMDA reporting?"

Frequently Asked Questions

Why can't I find my SOP with keyword search?

Try semantic search instead. It understands meaning, not just exact words. For example, searching "loan approval" will find SOPs about "underwriting decisions" and "credit authorization."

What happens if I change an SOP that other procedures depend on?

The system shows Impact Analysis when you view an SOP. This tells you exactly which other SOPs and processes will be affected. You can see these dependencies in the Graph Viewer or the right panel in Workspace.

Do I need to know YAML or metadata formats to create SOPs?

No. Just write your procedure in plain language. The AI generates all technical details, metadata, compliance tags, and formatting automatically. You only review and approve.

What's the difference between Atoms, Molecules, and Organisms?

Think of building with LEGO blocks. Atoms are individual bricks (forms, templates). Molecules are small assembled parts (simple processes). Organisms are complete models (full workflows). This prevents duplication - change an atom once, and all SOPs using it get updated.

How does the Graph Viewer help me?

It shows relationships visually. You can instantly see which SOPs depend on others, identify bottlenecks, and understand the full impact of changes. The different layouts help you explore from different perspectives.

What compliance frameworks does the system track?

The system automatically tags SOPs with relevant frameworks including FHA, TRID, RESPA, SOX, HMDA, AML/BSA, Fair Lending, and others. The AI identifies compliance requirements during SOP creation.

Can I export SOPs or integrate with other systems?

SOPs are stored in standard formats (Markdown, YAML, JSON) and can be exported. The system provides API endpoints for integration with other tools.

How do I know if an SOP is current?

Every SOP shows version number, last updated date, and status (Active, Draft, Archived). The Dashboard shows pending reviews. Set up notifications for SOPs you own or follow.
